checkout ( string  $uri,

Checkout a socket from the specified URI authority.

The resulting socket resource should be checked back in via SocketPool::checkin() once the calling code is finished with the stream (even if the socket has been closed). Failure to checkin sockets will result in memory leaks and socket queue blockage. Instead of checking the socket in again, it can also be cleared to prevent re-use.

Parameters
string$uriURI in scheme://host:port format. TCP is assumed if no scheme is present. An optional fragment component can be used to differentiate different socket groups connected to the same URI. Connections to the same host with a different ConnectContext must use separate socket groups internally to prevent TLS negotiation with the wrong peer name or other TLS settings.
ConnectContext$contextSocket connect context to use when connecting.
CancellationToken | null$tokenOptional cancellation token to cancel the checkout request.
