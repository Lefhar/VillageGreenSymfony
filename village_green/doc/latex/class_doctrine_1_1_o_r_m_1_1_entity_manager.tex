\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_entity_manager}{Entity\+Manager}} is the central access point to ORM functionality.

It is a facade to all different ORM subsystems such as \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_unit_of_work}{Unit\+Of\+Work}}, \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_query}{Query}} Language and Repository API. Instantiation is done through the static \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_entity_manager_ae65c152ffb3bcd53cfd8fba5c24d135a}{create()}} method. The quickest way to obtain a fully configured \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_entity_manager}{Entity\+Manager}} is\+: \begin{DoxyVerb}use Doctrine\ORM\Tools\Setup;
use Doctrine\ORM\EntityManager;

$paths = array('/path/to/entity/mapping/files');

$config = Setup::createAnnotationMetadataConfiguration($paths);
$dbParams = array('driver' => 'pdo_sqlite', 'memory' => true);
$entityManager = EntityManager::create($dbParams, $config);
\end{DoxyVerb}
 For more information see \mbox{\hyperlink{}{http\+://docs.\+doctrine-\/project.\+org/projects/doctrine-\/orm/en/latest/reference/configuration.\+html}}

You should never attempt to inherit from the \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_entity_manager}{Entity\+Manager}}\+: Inheritance is not a valid extension point for the \mbox{\hyperlink{class_doctrine_1_1_o_r_m_1_1_entity_manager}{Entity\+Manager}}. Instead you should take a look at the \{\begin{DoxySeeAlso}{See also}
\textbackslash{}\+Doctrine\textbackslash{}\+ORM\textbackslash{}\+Decorator\textbackslash{}\+Entity\+Manager\+Decorator\} and wrap your entity manager in a decorator. 
\end{DoxySeeAlso}

Gets a partial reference to the entity identified by the given type and identifier without actually loading it, if the entity is not yet loaded.

The returned reference may be a partial object if the entity is not yet loaded/managed. If it is a partial object it will not initialize the rest of the entity state on access. Thus you can only ever safely access the identifier of an entity obtained through this method.

The use-\/cases for partial references involve maintaining bidirectional associations without loading one side of the association or to update an entity without loading it. Note, however, that in the latter case the original (persistent) entity data will never be visible to the application (especially not event listeners) as it will never be loaded in the first place.
