\doxysubsection{Detailed Description}
\mbox{\hyperlink{class_monolog_1_1_handler_1_1_handler}{Handler}} to only pass log messages when a certain threshold of number of messages is reached.

This can be useful in cases of processing a batch of data, but you\textquotesingle{}re for example only interested in case it fails catastrophically instead of a warning for 1 or 2 events. Worse things can happen, right?

Usage example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\$log = \textcolor{keyword}{new} Logger(\textcolor{stringliteral}{'application'});}
\DoxyCodeLine{\$handler = \textcolor{keyword}{new} SomeHandler(...)}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// Pass all warnings to the handler when more than 10 \& all error messages when more then 5}}
\DoxyCodeLine{\$overflow = new OverflowHandler(\$handler, [Logger::WARNING => 10, Logger::ERROR => 5]);}
\DoxyCodeLine{}
\DoxyCodeLine{\$log-\/>pushHandler(\$overflow);}

\end{DoxyCode}

Handles a record.

All records may be passed to this method, and the handler should discard those that it does not want to handle.

The return value of this function controls the bubbling process of the handler stack. Unless the bubbling is interrupted (by returning true), the \mbox{\hyperlink{class_monolog_1_1_logger}{Logger}} class will keep on calling further handlers in the stack with a given log record.
