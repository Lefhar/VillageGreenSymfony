It now uses a versioned JSON format instead that is independent of PHP implementation details (see \href{https://github.com/sebastianbergmann/phpunit/issues/3581}{\texttt{ \#3581}} and \href{https://github.com/sebastianbergmann/phpunit/pull/4662}{\texttt{ \#4662}} for examples why this is a problem). When PHPUnit tries to load the test result cache from a file that does not exist, or from a file that does not contain data in JSON format, or from a file that contains data in a JSON format version other than the one used by the currently running PHPUnit version, then this is considered to be a \char`\"{}cache miss\char`\"{}. An empty {\ttfamily Default\+Test\+Result\+Cache} object is created in this case. This should also prevent PHPUnit from crashing when trying to load a test result cache file created by a different version of PHPUnit (see \href{https://github.com/sebastianbergmann/phpunit/issues/4580}{\texttt{ \#4580}} for example).
