\doxysubsection{Detailed Description}
Base HTTP client interface for use in \{\begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{interface_amp_1_1_http_1_1_client_1_1_application_interceptor}{Application\+Interceptor}}\}.
\end{DoxySeeAlso}
Applications and implementations should depend on \{\begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{class_amp_1_1_http_1_1_client_1_1_http_client}{Http\+Client}}\} instead. The intent of this interface is to allow static analysis tools to find interceptors that forget to pass the cancellation token down. This situation is created because of the cancellation token being optional.
\end{DoxySeeAlso}
Before executing or delegating the request, any client implementation must call \{\begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{interface_amp_1_1_http_1_1_client_1_1_event_listener_ab1fa25adc9bb0e933fb555cb0363c0d6}{Event\+Listener\+::start\+Request()}}\} on all event listeners registered on the given \mbox{\hyperlink{interface_amp_1_1_http_1_1_client_1_1_delegate_http_client_ae92be99a598a3a6249217209a9263c38}{request}} in the order defined by \{

\mbox{\hyperlink{class_amp_1_1_http_1_1_client_1_1_request_a6fcfc6d55c33b7665002e6245ca4f465}{Request\+::get\+Event\+Listeners()}}\}. Before calling the next listener, the promise returned from the previous one must resolve successfully.
