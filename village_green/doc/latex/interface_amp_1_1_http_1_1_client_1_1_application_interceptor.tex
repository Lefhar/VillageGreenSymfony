Intercepts an application request to an HTTP resource.

The implementation might modify the request, delegate the request handling to the {\ttfamily \$http\+Client}, and/or modify the response after the promise returned from {\ttfamily \$http\+Client-\/\texorpdfstring{$>$}{>}request(...)} resolves.

An interceptor might also short-\/circuit and not delegate to the {\ttfamily \$http\+Client} at all.

Any retry or cloned follow-\/up request must be manually cloned from {\ttfamily \$request} to ensure a properly working interceptor chain, e.\+g. the \{\begin{DoxySeeAlso}{See also}
Decompress\+Response\} interceptor only decodes a response if the {\ttfamily accept-\/encoding} header isn\textquotesingle{}t set manually. If the \mbox{\hyperlink{interface_amp_1_1_http_1_1_client_1_1_application_interceptor_a52e3bae929a024829dab8535eae7ec3e}{request}} isn\textquotesingle{}t cloned, the first attempt will set the header and the second attempt will see the header and won\textquotesingle{}t decode the response, because it thinks another interceptor or the application itself will care about the decoding.
